# 启动采集问题

用于 Skill 开始时主动问用户问题。目标是快速建立求职上下文，不是一次性审问。

## 提问原则

- 先收材料，再问缺口；两条线并行。
- 初始化目标是把已有材料整理成可复用素材，不是为了某一个岗位立刻裁剪材料。
- 默认每轮问 3 个以内；确实影响下一步判断时最多问 5 个，避免一次性问卷过载。
- 先问基础资料、已有材料、隐私边界和用户目标，再问细节。
- 用户答不上来的内容标记为“待补证据”，不要编。
- 如果用户已经给了材料，先读取材料，再只追问材料里缺失或冲突的内容。

## 第一轮最小问题

如果用户还没给足上下文，优先只问下面 3 个问题；用户答不全也可以先推进：

```markdown
1. 你现在有什么材料或线索？旧简历、岗位截图、作品链接、证书、项目文档、聊天描述都可以；没有材料也可以口头说。
2. 你这一轮最急想解决什么？例如改简历、找方向、拆 JD、准备投递、准备面试、谈薪，或者只是“不知道怎么开始”。
3. 有什么不能读取、不能公开、不能写进材料的内容？另外告诉我城市、岗位方向、时间线里最硬的限制即可。
```

第一轮之后，再根据目标补问学历、专业、毕业时间、可入职时间、证书、薪资期望、重点经历等信息。不要一开始把所有字段一次性抛给用户。

## 第二轮：经历与成果

```markdown
1. 你最近 1-3 段工作、实习、项目、校园活动或兼职分别做了什么？你的具体角色是什么？
2. 哪些成果有数字、截图、链接、文档或他人反馈可以证明？
3. 哪些成果你觉得重要，但目前还没有证据？
4. 你最能被追问讲深的项目是哪 2-3 个？
5. 面试官最可能质疑你什么？
```

## 学生/应届/实习补充问题

如果候选人是学生、应届生、实习求职者，补问：

```markdown
1. 你有哪些实习、兼职、社团、学生干部、志愿服务、比赛、科研、课程项目或毕业设计经历？
2. 有哪些获奖、奖学金、证书、语言成绩、专业排名或重要课程成绩？
3. 哪些经历能证明责任心、沟通、执行、学习速度、抗压或组织能力？
4. 有没有作品、报告、PPT、论文、代码、设计稿、活动照片、证书扫描件可以作为证据？
5. 你想投校招、实习、管培生、行业岗位，还是具体职能岗位？
```

## 非技术岗位补充问题

如果候选人不是技术/开发岗位，补问：

```markdown
1. 你主要服务过什么客户、用户、学生、病人、门店、渠道、供应商或内部团队？
2. 你处理过什么流程、活动、销售、运营、教学、服务、行政、人事、财务、内容或设计任务？
3. 你有没有交付量、转化、满意度、复购、成本、效率、活动规模、培训人数等结果？
4. 你最擅长的是沟通、执行、分析、组织、销售、审美、服务、管理，还是学习适应？
5. 你希望下一份工作继续做同类岗位，还是同岗位换行业/换城市/升职级？
```

## 第三轮：偏好与筛选

```markdown
1. 你更想要大公司、创业公司、外企、国企，还是无所谓？
2. 你更看重薪资、成长、稳定、自由度、行业、团队，还是岗位内容？
3. 你不想做什么类型的工作？
4. 你能接受出差、加班、驻场、销售性质、管理职责吗？
5. 你现在的求职时间线是什么？马上投、1 个月内、观望、只做材料准备？
```

## 第四轮：材料缺口

根据前面材料和回答，生成定制追问：

```markdown
| 缺口 | 为什么重要 | 需要用户补什么 |
| --- | --- | --- |
| 缺少项目/活动/工作截图 | 作品集和简历证据不足 | 提供 3-5 张核心流程截图或交付物 |
| 缺少成果数字 | 简历要点难以量化 | 确认用户数、收入、效率、交付量或时间节省 |
| 岗位目标模糊 | JD 定制无从排序 | 选择 2 个主投岗位族 |
```

## 输出格式

每轮问完后，更新：

```text
已确认信息
已有材料
待补材料
待确认问题
下一步建议
```

# 技能分层

用于把零散能力整理成求职技能地图。

## 通用分层模型

根据目标岗位调整名称：

```text
岗位核心技能：目标岗位最直接购买的能力
行业/业务技能：行业、客户、场景、流程、产品、服务对象
工具/方法技能：软件、设备、方法论、流程、专业工具、AI/数据工具
交付/执行技能：项目推进、活动执行、服务交付、质量控制、复盘
沟通/协作技能：表达、写作、销售、培训、协调、跨部门合作
成长/潜力信号：学习速度、抗压、责任心、领导力、主动性
```

如果候选人偏工程，技术层可以继续拆成：

```text
语言 / 框架 / 数据 / 基础设施 / 测试 / AI 技术栈
```

如果候选人偏非工程，要把技术能力和业务用途绑定：

```text
业务场景：销售、运营、营销、培训、服务、行政、人事、财务、设计、教育
工具/方法：Excel、PPT、CRM、ERP、剪辑、设计软件、数据分析、AI 工具、专业设备
结果信号：成交、转化、满意度、交付量、活动规模、培训人数、节省时间、降低成本
潜力信号：获奖、竞赛、奖学金、学生干部、社团组织、志愿服务、课程项目
```

## 映射模板

```markdown
# Skills Taxonomy

| 技能 | 分类 | 证据 | 岗位相关度 | 可信度 |
| --- | --- | --- | --- | --- |
|  | 岗位核心/行业业务/工具方法/交付执行/沟通协作/成长潜力 |  | 高/中/低 | 强/中/弱 |
```

## 规则

- 不列候选人无法面试解释的技能。
- 优先保留能出现在工作、实习、校园经历、项目、作品、获奖或目标 JD 里的技能。
- 只有当 JD 原词真实匹配候选人证据时，才使用 JD 原词。
- 简历 Skills 区保持紧凑；详细映射放在 taxonomy 文件里。

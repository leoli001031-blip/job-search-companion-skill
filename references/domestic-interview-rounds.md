# 国内分轮次面试

用于国内 HR 面、业务面、主管面、终面、敏感问题、谈薪前置准备。不要只用 STAR，应按面试轮次判断面试官真正筛什么。

## 面试轮次矩阵

```markdown
| 轮次 | 面试官在筛什么 | 候选人要证明什么 | 主要材料 |
| --- | --- | --- | --- |
| HR 初筛/电话面 | 沟通正常、基本匹配、风险、到岗、薪资、稳定性 | 表达清楚、动机合理、风险可控 | 30 秒自我介绍、求职动机、基本条件、风险口径 |
| 业务面 | 岗位能力、案例真实性、方法和结果 | 真做过、能复盘、能迁移到岗位 | 项目深挖、经历证据、JD 映射 |
| 主管面 | 协作方式、主动性、资源约束、成长潜力、长期匹配 | 能进团队、能落地、能承压 | 取舍故事、协作故事、公司理解 |
| 终面/Boss 面 | 价值观、成本收益、稳定性、岗位必要性 | 为什么值得招、为什么会留下 | 职业叙事、贡献点、反问 |
| HR 谈薪 | 薪资预期、预算匹配、offer 风险 | 有市场依据、有底线、能协商 | 薪资区间、整体包、入职条件 |
```

## 国内 HR 高频问题

准备这些问题时，目标不是“完美答案”，而是降低风险并保持真实。

```markdown
| 问题 | 面试官担心 | 回答结构 |
| --- | --- | --- |
| 为什么离职/换工作 | 是否抱怨、稳定性、真实动机 | 客观事实 -> 正向诉求 -> 目标岗位连接 -> 不攻击前公司 |
| 空窗期在做什么 | 能力是否断档、状态是否恢复 | 阶段安排 -> 做过的准备/项目/学习 -> 现在可投入 |
| 期望薪资 | 是否超预算、自我定位 | 市场/经验依据 -> 可接受区间 -> 关注整体包 -> 反问薪酬结构 |
| 到岗时间 | 排期风险 | 当前状态 -> 最早到岗时间 -> 交接/毕业/手续说明 |
| 能否接受加班/出差/驻场 | 岗位节奏和配合度 | 可接受关键节点投入 -> 重视效率 -> 确认实际频率 |
| 职业规划 | 是否短期跳槽、方向是否稳定 | 1-2 年能力目标 -> 和岗位积累一致 -> 不画过大的饼 |
| 你有什么缺点 | 自知力和改进意识 | 真实但非硬伤 -> 具体改进动作 -> 已有变化 |
| 手里有没有 offer | 入职概率和谈薪筹码 | 如实说明进度 -> 表达当前岗位优先级 -> 不虚构 offer |
| 背调/竞业/试用期 | 合规风险 | 如实说明 -> 可提供材料 -> 对不确定项先核验 |
```

## 敏感问题边界

婚育、家庭、性别、地域、年龄等与岗位无关的问题，不应成为能力评价依据。若被问到，优先温和转回岗位可用性：

```text
这个问题我理解公司可能是在确认岗位投入度。就岗位本身来说，我目前能保证 X 时间/地点/出差安排，也能按要求完成 Y 类工作。
```

不要主动透露无关隐私。若问题明显越界，先保持克制，记录风险，再决定是否继续流程。

## 线上面试 vs 线下面试

```markdown
| 项目 | 线上面试 | 线下面试 |
| --- | --- | --- |
| 设备 | 提前测试摄像头、麦克风、网络、电量 | 提前确认地址、交通、门禁、证件 |
| 背景 | 干净、安静、光线正面 | 着装和到场时间更重要 |
| 材料 | 屏幕旁放简历、JD、项目提示卡 | 带纸质简历、作品集、证件 |
| 眼神 | 看摄像头，不只盯屏幕 | 和面试官自然交流 |
| 突发 | 网络断开时立即文字说明并重连 | 迟到或变更及时电话/微信说明 |
```

微信视频面试也可能作为初筛方式。提前确认是否用微信、腾讯会议、飞书或其他软件，并测试昵称、头像和通知设置。

## 反问问题按轮次分类

```markdown
| 轮次 | 适合问什么 |
| --- | --- |
| HR 面 | 团队规模、招聘流程、入职时间、培养机制、薪酬结构大类 |
| 业务面 | 日常任务、关键指标、协作对象、当前项目难点 |
| 主管面 | 团队方向、岗位优先级、绩效标准、试用期目标 |
| 终面/Boss 面 | 公司阶段、岗位价值、长期发展、组织期望 |
```

多轮面试中，同一段经历、时间线和离职原因不要前后矛盾。每轮面完记录新信息，下一轮只补充，不推翻。

咨询、投资、管培生等岗位可能出现案例面试（Case Interview），需要单独练习结构化分析、假设拆解和口头推导；校招场景也可参考 `group-interview-campus.md`。

## 业务面项目深挖追问树

每个重点经历都要能回答：

```text
为什么做这个？
目标指标是什么？
你本人负责哪一块？
最难的约束是什么？
有哪些替代方案，为什么不用？
结果如何证明？
失败或不足是什么？
如果重做会改哪里？
这段经历如何迁移到本岗位？
```

## 公司理解表达分层

```text
30 秒 HR 版：我为什么关注贵司 + 岗位匹配点。
90 秒业务版：公开事实 + 岗位可能目标 + 我的相关证据。
追问版：我目前不确定 X，想请教团队现在更优先 A 还是 B。
```

实习、应届、基层岗位不要默认使用管理者视角的 30/60/90 天计划。改成：

```text
我会先熟悉什么
我会优先补齐什么
我能先交付什么
我需要向团队确认什么
```

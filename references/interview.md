# 面试准备

用于行为面试、项目深挖、技术面和转型解释。

国内求职不要只准备 STAR；HR 面、业务面、主管面、终面、敏感问题、谈薪前置见 `references/domestic-interview-rounds.md`。校招、实习、群面、测评、笔试见 `references/group-interview-campus.md`。

## STAR + Reflection

```text
Situation：背景和利害关系
Task：候选人承担的责任
Action：候选人具体做了什么
Result：结果、证据或学习
Reflection：这件事如何改变候选人的判断
```

## 故事库模板

```markdown
# 面试故事库

| 故事 | 能力点 | 来源项目/经历 | STAR 状态 | 证据 | 风险 |
| --- | --- | --- | --- | --- | --- |
```

## 必备故事

至少准备：

- 自我介绍。
- 为什么投这个岗位 / 这家公司。
- 职业转型解释。
- 一个困难问题。
- 一个取舍或优先级决策。
- 一个失败和修正。
- 一个协作或利益相关方沟通故事。
- 一个项目深挖。
- 一个学习速度故事。
- 一个公司理解与岗位贡献故事。

## 项目深挖问题

每个重点项目准备：

```text
解决了什么问题？
为什么选择这个方案？
你本人具体做了什么或决定了什么？
最难的地方是什么？
下一步会怎么改进？
它如何映射到目标岗位？
```

不要伪装确定性。如果证据不完整，就准备清楚说明当前状态和下一步。

## 技术面试

技术相关岗位常见形式：

```markdown
| 形式 | 准备重点 |
| --- | --- |
| 算法题 | 高频题型、边界条件、复杂度说明 |
| 系统设计 | 需求澄清、架构取舍、容量估算、瓶颈分析 |
| 项目深挖 | 真实角色、技术难点、方案取舍、线上问题 |
| 八股/基础题 | 语言、框架、数据库、网络、操作系统等基础 |
| 白板/共享屏编码 | 先讲思路，再写代码，边界测试，最后复盘 |
```

白板编程 6 步：

```text
澄清题意 -> 说出思路 -> 写主流程 -> 补边界 -> 走样例 -> 分析复杂度
```

刷题不要只背答案。优先按目标岗位和公司历史题型建立题单。

## 面试紧张管理

- 提前录屏排练 30 秒自我介绍和 2 个项目故事。
- 把面试看成双向确认，不是单方面被审判。
- 遇到不会的问题，先复述理解和思路，不要沉默或硬编。

## 公司理解与岗位贡献

面试前把公司调研转成可表达材料：

```markdown
# 公司理解回答

## 为什么是这家公司

## 我对公司/业务的一个具体观察

## 这个岗位可能要解决的问题

## 我的经历如何对应

## 如果入职，会先熟悉/补齐/交付什么

## 我想问面试官的问题

## 不建议说的话
```

表达要克制。可以说“我观察到”“我理解可能是”“我会先验证”，不要把公开信息推断说成内部事实，也不要直接批判公司战略。

如果候选人是实习、应届或基层岗位，不要默认讲管理者式 30/60/90 天计划；改成“先熟悉业务和流程、补齐岗位工具/方法、先交付一个明确任务”。

如果公司公开信息很少，诚实说明信息有限：

```text
我目前能查到的公开信息有限，所以不会过度判断公司内部情况。
基于 JD，我理解这个岗位可能重点涉及 X。
我想进一步请教，这个岗位当前最优先解决的是 A 还是 B？
```

信息不足时，不要为了显得准备充分而编造公司背景、业务问题或近期动态。

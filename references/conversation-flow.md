# 对话流程与引导

用于控制求职陪跑的对话节奏，避免一次性信息过载。内部可以完整分析，对话只推进用户当前最需要完成的一步。

## 核心原则

- 每轮只推进一个主要动作：收材料、澄清目标、拆 JD、补证据、确认简历结构、出材料、准备面试、投递跟进、复盘。
- 先给用户可执行的下一步，再按需展开完整表格。
- 不把内部审查语言、完整评分表、证据矩阵、公司调研长表和量化补采表同时甩给用户。
- 用户可以混乱、跳步、反悔、只给一点信息；先帮助收敛，不要求用户一次性填完整问卷。
- 每轮都要让用户知道：现在在哪、为什么做这一步、他只需要做什么、做完会得到什么。
- 不把外部结果写成可保证事项。对话只围绕可控动作推进：整理事实、判断匹配、产出材料、记录反馈、安排下一步。

## 陪跑进度条

当用户问“现在到哪了 / 下一步是什么 / 我该怎么用这个 skill”时，用这条进度定位，不直接展开完整工作流。

```text
0 接住混乱：先听懂用户状态和现有材料
1 素材池：整理经历、证据、缺口和可迁移能力
2 方向/JD：用目标方向或具体 JD 判断岗位在考什么
3 最小可投材料：先做能投、能解释、能追问的版本
4 小批量投递：用少量岗位测试反馈，不盲目海投
5 面试准备：围绕 JD 追问、项目细节和公司理解做准备
6 反馈复盘：把无反馈、追问、失败和新信息回流到材料
7 Offer/入职检查：确认薪酬、合同、背调、试用期和风险
```

每轮最多暴露当前步、下一步和关键风险；不要把 0-7 全部展开成大报告。

## 零说明启动原则

不要求用户先阅读 README 或理解完整流程。对话里直接完成引导：

- 不要求用户先看安装说明、工作流或材料清单。
- 用户只发一句“我想找工作/我很乱/帮我改简历”时，直接进入快速诊断或素材接收。
- 用户只发 JD、截图、旧简历或文件夹时，先说明“我已经能从这些开始”，再列不超过 3 个最影响下一步的问题。
- 先给行动版；完整流程、评分表、证据矩阵和长报告只在用户要求、人工复核或文件交付时展开。

## 行动卡

普通对话默认使用这个结构：

```markdown
## 当前阶段

## 当前判断
- [1-3 个关键判断]

## 你现在只需要做
1. [最小动作]
2. [可选动作]
3. [可选动作]

## 下一步
做完后，我会进入 [下一阶段]，产出 [下一个可用材料]。
```

如果当前任务涉及投递、面试或失败反馈，追加一个轻量复盘点：

```markdown
## 这轮反馈要记录
- [投递回复 / HR 追问 / 面试问题 / 用户状态 / 新发现]
```

如果已有可直接使用内容，可以加一个短小模块：

```markdown
## 可直接使用

[话术 / bullet / 简历段落 / 面试回答]
```

## 阶段状态机

按用户当前状态选择阶段，不强行从第 0 步跑到最后。

| 用户状态 | 当前阶段 | 对话只输出 | 完成标准 | 下一阶段 |
| --- | --- | --- | --- | --- |
| 很迷茫、没材料 | 快速诊断 | 3 个问题 + 2-3 个可能方向 | 用户回答最小问题 | 素材接收 |
| 发来旧简历/文件夹/零散经历 | 素材接收 | 已读到什么、缺什么、先补哪 3 个 | 形成基础素材池 | 目标澄清 |
| 有多个方向 | 目标澄清 | 方向对比、投入成本、最小试投建议 | 用户选定 1-2 个方向 | JD 样本拆解 |
| 发来具体 JD | JD 作答 | 投递判断、3-5 道 JD 题、强证据、最大缺口 | 明确投/暂缓/跳过和补强动作 | 公司理解或材料定制 |
| 重要岗位/有内推 | 高价值投递 | 公司理解卡、主动提案、最小投递包 | 有定制简历/作品/短话术 | 投递沟通 |
| 要写简历 | 简历编排确认 | 2-3 种结构方案、推荐方案、取舍建议 | 用户确认重点和顺序 | 简历正文 |
| 简历已出 | 简历使用 | 可用等级、3 个短板、发送前动作 | 用户知道是否可投 | PDF/投递/面试 |
| 要做 PDF | PDF 模板确认 | ATS/侧栏/照片/作品版选择和风险 | 用户确认模板和主题 | PDF 生成 |
| 要准备面试 | 面试准备 | 当前面试目标、最可能追问、3 个故事 | 用户确认轮次和岗位 | 模拟面试 |
| 投递无反馈/失败 | 复盘 | 可能原因、需要改的 1-3 件事 | 形成下一轮调整 | 简历/JD/话术迭代 |

## 反馈回流循环

投递和面试后的反馈必须回流到下一轮行动，不要只安慰或只催继续投。

```text
收到反馈
↓
先分类：岗位匹配 / 材料呈现 / 面试表达 / 渠道时机 / 用户状态
↓
只选 1-3 个可改变量
↓
更新简历、话术、作品、岗位池或面试故事
↓
小批量再测试
```

默认复盘口径：

- 没有回复是低信息量信号，不自动代表用户不行。
- 面试失败先看证据、表达、匹配和市场因素，不评价用户人格。
- 连续出现同一问题时，把它转成补证据、模拟项目、学习附录或话术重写任务。
- 用户状态明显变差时，先缩小行动范围，不继续加任务。

## 各阶段模板

### 快速诊断

```markdown
## 当前阶段
快速诊断

## 我先不急着写简历
现在需要先知道你有什么、想去哪、最不想要什么。

## 你现在只需要回答
1. 过去 2 年你花最多时间做的 3 件事是什么？
2. 哪件事最常被别人认可或找你帮忙？
3. 你最不想做什么类型的工作？

## 下一步
答完后，我会提取可迁移能力，并给 2-3 个方向供你判断。
```

### 素材接收

```markdown
## 当前阶段
素材接收

## 我已经看到
- [材料 1]
- [材料 2]

## 现在最缺
- [影响下一步的缺口，不超过 3 个]

## 你现在只需要做
1. 补 [最关键材料/事实]
2. 确认 [目标/城市/时间线]

## 下一步
我会把这些整理成可复用素材池，再决定先做简历还是先拆 JD。
```

### JD 作答

```markdown
## 当前阶段
JD 作答

## 当前判断
- 投递建议：[立即投递/带解释投递/暂缓/跳过]
- 这份 JD 最像在考：[3-5 道题]
- 你当前最强证据是：[1-3 个]
- 当前最大缺口是：[1-3 个]

## 你现在只需要做
1. [补一个事实/材料]
2. [做一个小作品/确认一个口径]
3. [决定是否继续定制]

## 下一步
确认后，我会生成对应的简历策略、作品说明或投递话术。
```

### 简历编排确认

```markdown
## 当前阶段
简历编排确认

## 我建议先确认重点
这一步不是写正文，而是决定简历从上到下怎么排。

## 可选结构
1. [方案 A：适合什么，风险是什么]
2. [方案 B：适合什么，风险是什么]
3. [方案 C：适合什么，风险是什么]

## 推荐方案
[推荐一个，并说明一句理由]

## 你现在只需要做
确认用 A/B/C，或告诉我你想突出什么。
```

### 成品交付

```markdown
## 当前阶段
成品交付

## 可直接使用
[简历正文 / 话术 / 面试回答 / PDF 路径]

## 当前可用等级
[绿/黄/红]：[一句话判断]

## 发送前只需要检查
1.
2.
3.

## 下一步
[投递 / PDF / 面试准备 / 复盘]
```

## 长表展开规则

默认不展开完整表格。只有以下情况才展开：

- 用户明确说“完整展开 / 给我表格 / 保存成文件 / 帮我复核”。
- 当前任务是人工复核或交付报告。
- 高风险材料如果不展开会导致用户误用。
- 用户已经确认要做成工作文件。

展开前先提示：

```text
这部分会比较长。我先给行动版；如果你要，我再展开完整表格。
```

## 用户跳步处理

用户跳步时，不要僵硬阻止，也不要完全绕过必要确认。

| 用户说法 | 处理方式 |
| --- | --- |
| “直接帮我写简历” | 可以给草稿，但先用 3-5 句确认重点、风险和待确认项；不要声称终稿。 |
| “你帮我编一下” | 拒绝编真实经历，改为模拟练习/个人项目/作品样例路径。 |
| “这个不要写” | 判断是偏好还是求职效果问题；高风险时明确不建议。 |
| “我现在很乱” | 回到快速诊断，只问 3 个问题。 |
| “我换方向了” | 做目标变化确认：沿用什么、重做什么、先不动什么。 |
| “全部给我” | 可以展开完整报告，但先提醒信息量大，并建议同时给行动版。 |

## 上下文恢复摘要

多轮对话、长任务或用户离开后回来时，用 4 行恢复上下文：

```markdown
## 当前求职状态
- 目标：
- 已确认：
- 待确认：
- 下一步：
```

触发时机：

- 用户说“继续 / 回到刚才 / 下一步 / 现在到哪了”。
- 对话超过 8-10 轮且信息变多。
- 用户更换 JD、公司、简历版本或交付物。
- 生成重要成品前。

## 交付节奏建议

不要把“完整流程”理解成一次性输出所有产物。更稳的节奏是：

```text
第 1 轮：收材料和目标
第 2 轮：JD/方向判断
第 3 轮：补证据或做小作品
第 4 轮：确认简历结构
第 5 轮：出简历正文
第 6 轮：出 PDF/投递话术
第 7 轮：面试准备
第 8 轮：投递/面试反馈复盘
```

用户急投时可以压缩，但仍要保留“当前状态 + 最小确认 + 风险提示 + 下一步”。

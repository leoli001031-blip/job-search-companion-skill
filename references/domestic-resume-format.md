# 国内简历格式与照片文化

用于判断中文简历是否放照片、教育背景位置、政治面貌、籍贯、期望薪资、自我评价和一页/两页取舍。

## 结构决策

传统投递 / 平台友好：

```text
基本信息
求职意向
教育背景
个人优势
专业技能
项目 / 工作 / 实习经历
校园 / 证书 / 作品链接
```

作品型 / 创意型：

```text
基本信息
求职意向
个人定位
项目 / 作品
相关经历
技能
教育背景
作品集链接
```

判断规则：

- JD 明确学历/专业/证书：教育背景靠前。
- 用户是应届/实习/管培：教育背景靠前。
- 用户是社招且经历强：工作经历靠前。
- 用户是转型候选人：教育前置 + 项目前置，先过门槛，再证明能力。

## 照片要不要放

建议放照片：

- 国企/事业单位/传统企业。
- 服务业、销售、前台、讲师、顾问、需要强线下形象岗位。
- 招聘平台字段明确要求。

建议不放照片：

- 互联网技术岗、研发、算法、纯产品、外企。
- 用户没有合适职业照片。
- 照片会降低专业感。

如果放：

- 使用真实职业照片，不建议从零生成虚构照片。
- 背景干净，五官清晰，着装简单。
- 不用生活照、自拍、过度美颜图。

## 政治面貌 / 籍贯 / 户籍

政治面貌：

- 国企、事业单位、公务员、部分校招可以写。
- 外企、互联网、普通民企一般不写。

籍贯/户籍：

- 岗位或城市明确关注稳定性、落户、通勤时可写。
- 无关时不主动写。

## 自我评价

国内简历可以有“个人优势/自我评价”，但要短。

推荐：

```text
3-4 条，围绕岗位购买点：行业经验、项目证据、工具能力、沟通协作。
```

避免：

```text
性格开朗、吃苦耐劳、责任心强、学习能力强。
```

除非后面有具体经历证明。

## 期望薪资

在线平台常要求填写。

策略：

- 不确定市场价：写区间或面议。
- 已有明确底线：写合理区间，不要只写最低线。
- 附件 PDF 简历一般不主动写期望薪资。

## 一页还是两页

- 应届/实习：优先 1 页。
- 项目型转型候选人：2 页可以接受，但第一页必须给出岗位匹配和核心项目。
- 作品很多：简历控制 1-2 页，作品集另附链接。

## 国内常用简历工具

- Word / WPS：最稳，适合传统企业和需要可编辑版本的场景。
- Markdown / HTML -> PDF：适合技术、产品、内容类候选人，便于版本管理。
- 超级简历、知页简历等在线工具：适合快速出标准版，但要检查导出的 PDF 是否可复制文本。
- Canva 等设计工具：适合视觉岗位，但不要牺牲招聘系统解析可读性。

## 中文排版建议

- 字体：微软雅黑、黑体、宋体等常见字体。
- 正文字号：10.5-12pt。
- 行距：1.2-1.5。
- 页边距：约 1.5-2cm。
- 标题层级不超过 2-3 层。

## 招聘系统解析注意事项

国内招聘系统可能使用北森、Moka、大易等系统，或平台自有解析。通用建议：

- 单栏布局优先。
- 避免复杂表格、文本框、图标和图片化文字。
- 公司、学校、岗位、时间不要写在图片里。
- 导出 PDF 后测试能否选中文本。
- 在线平台期望薪资是否写“面议”要看平台机制；不确定时写合理区间更稳。

## 格式禁忌

- 不用复杂表格、文本框和图形堆叠影响系统解析。
- 不把联系方式放页眉图片里。
- 不用扫描版 PDF。
- 中文简历少用夸张形容词，多写场景、动作、交付物。
- 邮箱建议使用 Outlook、163、126、foxmail 或姓名拼音相关邮箱；避免纯数字 QQ 邮箱。如果只有 QQ 邮箱，可考虑使用 foxmail 别名，让观感更职业。

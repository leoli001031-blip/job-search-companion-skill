# 交付物评分与使用引导

用于给简历、项目案例、JD 匹配、公司调研、面试故事、投递话术、谈薪清单等求职产物做成熟度评分、短板诊断和下一步强化建议。

## 核心原则

- 评分对象是“材料成熟度”和“岗位匹配度”，不是评价用户本人。
- 分数必须附理由；材料缺失时标注不确定性，不要假装精确。
- 产物不是终稿。每次输出都要告诉用户：怎么用、当前够不够投、哪里还弱、先补哪三件事。
- 评分要帮助行动，不制造焦虑。低分意味着“下一步更清楚”，不是“用户不行”。
- 不要因为分数低就停止推进；先给当前可用版本，再列补强路径。
- 交付前做通篇复核，不只检查固定项，也主动寻找仍未解决的问题、前后矛盾、风险缺口和用户可能误用之处。

## 默认交付结构

重要产物默认附 4 个模块，并在结尾附通篇复核：

```markdown
## 怎么使用

## 当前评分

## 主要短板

## 下一步补强
```

如果产物是简历、项目案例或作品集，还要附：

```markdown
## 成果量化补采
```

交付结尾用一句或几句完成通篇复核，不需要展开成长报告。

## 轻量交付物例外

如果用户只要求一句话术、一个标题、一个简历 bullet、一个岗位判断、一个追问回复或一次局部改写，不强制输出完整评分表。轻量交付只需要：

```markdown
## 可直接使用

## 使用提醒

## 仍需确认
```

如果轻量任务涉及最终投递、PDF 文件、薪资、背调、敏感问题或明显证据风险，仍要补一句风险提示和下一步动作。

## 通篇复核

终稿、可投递版、PDF、投递话术、面试口径和重要评估产物输出前，先做残留问题扫描。不要只按固定清单打勾；还要主动找：

- 身份、路径、学校、公司、JD、文件名是否混入示例或历史候选人信息。
- 文件状态和导出状态是否真实；未写入验证时是否误称“已生成”。
- 数字、成果词、动作词是否有证据，是否存在越界包装。
- 简历结构、PDF 模板、投递话术或面试口径是否匹配用户目标、行业和使用场景。
- 是否有前后矛盾、未确认事实、风险未解释、下一步不清或用户可能误用的地方。

输出格式保持轻量：

```text
通篇复核：已扫一遍，当前仍需注意：1）... 2）... 3）...
通篇复核：未发现会影响本轮使用的重大问题；待确认项已放入下一步补强。
```

如果本轮只是草稿，也要说明草稿层面的残留问题，避免用户误当终稿使用。

## 评分等级

优先给红黄绿判断，再按需要给数字。数字用于材料复盘，不用于评价用户本人。

```text
85-100：可直接使用。只做小修。
70-84：可以使用，但建议边投边补。适合“带解释投递”。
55-69：风险较明显。可用于内部讨论或素材整理，不建议直接大规模投递。
0-54：当前产物不适合直接投递或对外发送。先补目标、证据、结构或岗位匹配。
```

红黄绿判断：

- 绿：当前可用，适合直接投递、发送或练习。
- 黄：能用，但需要明确解释、补证据或小批量测试。
- 红：直接使用风险高，容易误导、被追问或降低匹配度；先做内部讨论或重写。

## 简历评分表

总分 100。适用于附件简历、平台简历、定制简历。

```markdown
| 维度 | 分值 | 评分 | 判断标准 | 需要加强 |
| --- | ---: | ---: | --- | --- |
| 岗位匹配 | 20 |  | 求职意向、关键词、核心项目是否贴 JD |  |
| 证据真实性 | 20 |  | 经历是否可证明、可追问、角色边界清楚 |  |
| 成果量化 | 15 |  | 是否有人数、次数、金额、效率、规模、反馈或可展示结果 |  |
| 国内格式与 ATS | 15 |  | 教育/经历顺序、单栏文本、时间线、平台友好 |  |
| 求职叙事 | 15 |  | 转型逻辑、个人优势、经历选择是否自然 |  |
| 风险控制 | 10 |  | 空窗、非相关专业、短经历、夸大表述是否处理 |  |
| 可读性 | 5 |  | 一屏重点、语言简洁、无模板废话 |  |
| 总分 | 100 |  |  |  |
```

简历短板诊断优先级：

1. 岗位匹配不清：先改求职意向、个人优势和项目选择。
2. 证据不足：先补截图、链接、证书、项目说明、作品页。
3. 成果不量化：先补成果量化补采表。
4. 国内格式不稳：调整教育背景位置、时间线、平台关键词和 ATS 友好格式。
5. 风险没解释：准备 30 秒和 2 分钟口径，不在简历中过度辩解。

简历使用说明要回答：

- 这版适合投什么岗位？
- 不适合投什么岗位？
- 是否能直接发 HR？
- 是否需要同步更新平台在线简历？
- 发送前还要补什么最小材料？

## 面试故事评分表

```markdown
| 维度 | 分值 | 评分 | 判断标准 | 需要加强 |
| --- | ---: | ---: | --- | --- |
| STAR 完整 | 20 |  | 情境、任务、行动、结果是否完整 |  |
| 角色边界 | 20 |  | 候选人做了什么，团队做了什么，是否清楚 |  |
| 结果证据 | 20 |  | 是否有数字、交付物、反馈或复盘 |  |
| 岗位相关 | 20 |  | 是否对应 JD 能力点，而不是只讲经历 |  |
| 追问韧性 | 10 |  | 面试官深挖时能否讲细节 |  |
| 表达自然 | 10 |  | 是否像真人经历，不像背稿 |  |
| 总分 | 100 |  |  |  |
```

低分处理：

- STAR 不完整：先补任务和结果。
- 角色不清：改动词强度，区分负责/参与/协助。
- 没结果：补交付物、反馈、学习或复盘。
- 不相关：换故事或重写能力点。

## 人脉私信

```text
你好 [Name]，我正在了解 [role/domain] 方向，看到你在 [company/team] 的经历。
我特别想了解 [specific reason]。
如果方便的话，能否请教你 1 个问题：这个岗位实际最看重什么能力？
```

保持低压力。不要一上来就让陌生人帮忙找工作。

## 跟进

这些情况可以跟进：

- 招聘方在合理时间内没有回复。
- 新证据能增强申请。
- 面试后需要发送感谢和补充信息。

## 投递去重

同公司同岗位不要在多个渠道乱投。跟踪表里记录：

```text
公司、岗位名、岗位编号、渠道、简历版本、推荐人/猎头、投递日期
```

如果已经通过内推或猎头进入流程，再从官网或 BOSS 重复投递前，先确认是否会冲突。

## 面试复盘模板

面完 30 分钟内记录：

```markdown
| 项目 | 事实记录 | 情绪标签 | 下一步 |
| --- | --- | --- | --- |
| 答得好的 3 点 |  | 自信/平静/意外 | 保留并强化 |
| 答得不好的 3 点 |  | 紧张/混乱/羞耻 | 补材料或补回答 |
| 新出现的问题 |  | 意外/焦虑 | 加入问题库 |
| 面试官透露的信息 |  | 希望/失望/不确定 | 调整公司理解 |
| 下一轮要补的材料 |  | 有把握/没把握 | 明确补证据动作 |
| 我的整体状态 |  | 亢奋/正常/低落 | 判断是否需要休息 |
```

## 面试探行业与作品迭代

面试不只用于争取录用，也可以用于了解行业真实要求。一次失败面试如果能带回判断标准、常见追问和材料短板，就不是纯损耗。

复盘时追加：

```markdown
| 复盘项 | 记录 | 下一步动作 |
| --- | --- | --- |
| 面试官真正关心的问题 |  | 加入 JD 题目库 |
| 被追问最深的经历/作品 |  | 强化证据、补截图、补数字 |
| 对方不买账的表达 |  | 改写简历 bullet 或面试说法 |
| 新了解到的岗位标准 |  | 更新目标岗位要求地图 |
| 可以转成作品集的反馈 |  | 替换低分作品页或新增案例页 |
```

迭代规则：

- 不把一次拒绝直接解释成“我不行”；先判断是岗位不匹配、证据不够、表达不清，还是作品没有回答 JD。
- 如果连续 3 次被问到同一缺口，必须转成学习附录、模拟作品或简历重写任务。
- 如果面试官对某个作品提出有价值反馈，优先更新作品集，而不是只改自我介绍。
- 不用攻击性“踢馆”表达。可以做前置思考、改进提案或模拟方案，但对外说法要克制、请教式、尊重对方已有工作。

## 情绪复盘衔接

如果用户出现投递无反馈、连续面试失败、自我怀疑、空窗期焦虑或明显求职疲惫，同时读取 `emotional-resilience.md`。

复盘时区分两类问题：

```text
工具型问题：简历、话术、项目证据、岗位池、面试回答。
状态型问题：焦虑、羞耻、自我怀疑、能量过低、持续回避。
```

如果连续多次复盘都出现“低落/羞耻/想放弃”，优先建议短暂停止投递，恢复节奏后再做策略调整。

## 背调准备

背调可能核验：

- 学历、工作时间、岗位名称。
- 离职原因、绩效或工作表现。
- 犯罪记录、商业冲突、竞业限制。
- 薪资证明或流水，视公司要求而定。

准备：

- 确认简历时间线和社保/离职证明大体一致。
- 提前准备学历证明、离职证明、推荐人信息。
- 有空窗、短经历、岗位名称不一致时，提前准备解释口径。
- 薪资流水涉及隐私，优先确认用途、接收主体和是否可脱敏。

## Offer 评估

比较：

```text
岗位匹配
直属上级/团队
学习曲线
薪酬
稳定性
地点/远程
成长路径
风险
```

接受前应拿到书面条款。不要只凭口头 offer 做最终决定。

口头 Offer 也有时效。拿到后确认：

```text
书面 Offer 预计何时发？
回复截止日期是什么时候？
入职日期是否可协商？
未确认薪资/试用期/社保公积金前，不要急着口头承诺。
```

## 劳动关系主体检查

国内 Offer 不能只看岗位名和薪资，还要确认“和谁签、谁发薪、在哪工作、归谁管理”。

```markdown
| 项目 | 要确认什么 | 风险 |
| --- | --- | --- |
| 签约主体 | 劳动合同和哪家公司签 | 招聘品牌与签约主体不一致 |
| 发薪主体 | 工资由谁发放，个税由谁申报 | 第三方发薪、薪资证明不一致 |
| 社保主体 | 社保/公积金在哪个城市、由谁缴纳 | 到手、落户、购房、医保受影响 |
| 工作地点 | 自有办公、客户现场、门店、驻场 | 实际通勤和工作内容漂移 |
| 管理关系 | 直属上级属于哪家公司/部门 | 外包或派遣岗位成长受限 |
| 合同类型 | 劳动合同、劳务合同、实习协议、派遣、外包 | 权益和解除规则不同 |
```

如果出现“外包、派遣、驻场、第三方、人力资源公司代签、项目合同、劳务合作”等词，不直接否定，但要把岗位稳定性、社保、转正机会、绩效归属和简历呈现口径问清楚。

## 背调授权边界

背调前先确认：

```text
背调公司/接收主体是谁？
背调范围包括哪些内容？
是否需要本人书面授权？
是否会联系现任/前任公司？联系前是否会告知？
薪资流水、社保记录、离职证明是否可脱敏？
```

未获得明确授权和流程说明前，不建议主动发送薪资流水、完整身份证件、完整家庭信息或无关隐私材料。

## 谈薪与 Offer 确认清单

```markdown
| 项目 | 要确认什么 | 风险 |
| --- | --- | --- |
| base | 月薪、发薪月数、试用期是否折扣 | 口头月薪和 offer 不一致 |
| 绩效/提成 | 计算方式、发放周期、达成条件 | 只说“综合薪资”不说结构 |
| 年终奖 | 几薪、是否保底、入职未满一年怎么算 | 写法模糊 |
| 社保公积金 | 缴纳基数、比例、城市 | 按最低基数缴纳 |
| 试用期 | 时长、工资、转正标准 | 试用期过长或标准不清 |
| 工作地点 | 城市、办公点、驻场/出差 | 实际地点变化 |
| 工作时间 | 排班、加班、调休、远程 | 只说“弹性”不说边界 |
| 汇报关系 | 直属上级、团队规模、岗位归属 | 岗位职责漂移 |
| 背调 | 时间、范围、授权材料 | 未提前解释空窗/离职 |
| 竞业/保密 | 是否签署、范围、补偿 | 约束过宽或无补偿 |
| 三方/网签 | 签约主体、违约条款、学校流程 | 应届生毁约成本 |
```

谈薪口径：

```text
我对这个岗位是有兴趣的。结合岗位要求、所在城市和我的经历，我期望整体薪酬在 X-Y 区间。也想进一步了解贵司薪酬结构，包括 base、绩效/年终、试用期和社保公积金口径。
```

接受 offer 前输出：

```text
已确认条款
待确认条款
红旗风险
是否建议接受
下一步沟通话术
```

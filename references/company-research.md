# 公司调研与业务理解

用于具体公司、具体 JD、面试准备、求职信、内推私信和 offer 判断。

## 什么时候做

在这些场景必须做公司调研：

- 用户提供了具体公司或具体 JD。
- 要写求职信、内推私信、面试自我介绍或“为什么选择我们公司”。
- 要准备“你了解我们公司吗”“你觉得这个岗位能带来什么价值”。
- 要判断岗位是否值得投、是否有业务风险或发展风险。

如果公司背景、产品、管理层、融资、财报、新闻或岗位状态可能变化，必须联网核验；无法核验时标注“未实时核验”。如果可靠信息很少，诚实标注“信息不足”，不要补编公司背景。

## 默认输出形态：公司理解卡

除非用户明确要求深度调研报告，默认不要输出大篇公司研究。具体 JD 场景优先给一张轻量卡：

```markdown
## 公司理解卡

- 能确定说的：
- 合理推断：
- 对这个岗位意味着什么：
- 候选人可以怎么连接：
- 面试 30 秒说法：
- 可以反问：
- 不要说：
- 信息来源与不确定性：
```

公司理解卡用于后续 JD 匹配、简历策略、BOSS/微信话术和面试表达。内部可保留更完整调研，但用户默认只需要可使用的结论。

内部调研要足，对外表达要收：可以检查多个来源、保存链接和不确定性，但对用户默认只展示能支撑行动的事实、推断、可说口径和反问问题。

## 信息来源优先级

优先使用：

```text
公司官网 / 产品页 / 招聘页
官方公众号、新闻稿、公告、财报、招股书、投资者关系页面
可信媒体报道、行业报告、监管/工商/交易所公开信息
产品实际体验、App 商店、用户评价、公开案例
招聘平台上的同公司相似岗位 JD
员工评价、社区帖子、匿名爆料只作风险提示，不当作事实
```

国内求职可追加非官方情报源：

```text
脉脉：公司氛围、部门变动、内部员工视角、薪资爆料线索
牛客：校招/互联网面经、笔试题、HR 面问题、薪资开奖线索
看准网/职友集：面试难度、薪资区间、公司评价线索
小红书/知乎：岗位体验、离职复盘、行业体感
```

这些来源必须交叉验证。不要在面试中直接说“我看脉脉/小红书上说你们……”。具体方法见 `domestic-intelligence-sources.md`。

输出时区分：

```text
事实：来源能支持。
推断：由多个事实推出来，但公司未明确说。
候选人观点：用于面试表达的个人判断。
待验证：需要面试或进一步资料确认。
信息不足：公开信息太少，不能形成可靠判断。
```

```markdown
- 核心产品/服务：
- 主要客户/用户：
- 商业模式：

- 新产品/新业务：

- 市场动作：
- 重要新闻：

## 岗位上下文
- 这个岗位服务哪个业务/团队：
- 可能负责什么问题：
- 上游和下游是谁：
- 可能看什么结果指标：

## 可能的问题或挑战
- 从公开信息能看出的挑战：
- 从 JD 能推断出的挑战：
- 需要面试验证的问题：

## 候选人连接点
- 候选人哪些经历能对应公司当前需求：
- 候选人能贡献的具体价值：
- 需要克制表达的短板：

## 面试表达素材
- 为什么关注这家公司：
- 对公司/业务的一个具体观察：
- 自己能帮上的地方：
- 想进一步请教的问题：
```

## 面试表达原则

好的表达不是“我来给公司诊断问题”，而是：

```text
我做过调研
我理解这个岗位可能服务的业务目标
我能把自己的经历连接到这个目标
我知道仍有信息需要向面试官确认
```

按候选人层级调整表达：

```text
实习/应届/基层：我会先熟悉业务、补齐工具方法、完成明确交付。
社招熟手：我会先确认目标和约束，再复用过往方法解决局部问题。
管理/专家：可以谈 30/60/90 天判断，但必须标注需要入职后验证。
```

如果查不到足够可靠信息，好的表达是：

```text
我目前能查到的公开信息比较有限，所以不想做过度判断。
基于 JD，我理解这个岗位至少会涉及 X 和 Y。
我比较想在面试中进一步确认：这个岗位当前最核心的目标是 A，还是 B？
如果是 A，我过去的 C 经历可能能提供帮助；如果是 B，我会先补充了解 D。
```

推荐口径：

```text
我看到贵司近期在 X 方向有动作，结合 JD 里提到的 Y，我理解这个岗位可能需要解决 Z 类问题。
我过去在 A 经历中做过 B，虽然场景不同，但其中的 C 能力可以迁移到这里。
如果进入团队，我会先熟悉/验证 D，再从 E 这个具体切入点开始推进。
```

避免口径：

```text
你们公司最大的问题就是……
我认为你们应该立刻……
我可以解决你们的增长/管理/产品问题。
我看网上说你们……
虽然我没查到，但我猜贵司……
```

## 公司问题分析的边界

可以分析：

- JD 暗示的业务压力，例如增长、交付、客户支持、内容供给、运营效率、销售转化、产品体验。
- 公开信息显示的方向变化，例如新业务、出海、降本增效、组织扩张、产品升级。
- 候选人可以参与解决的局部问题。

不要做：

- 基于少量信息武断批判公司战略。
- 讨论未经证实的负面传闻。
- 把自己包装成能解决公司核心战略问题的人。
- 对自己不了解的财务、组织、技术内幕做确定判断。
- 在公司信息不足时，为了显得准备充分而编造背景、竞品、业务问题或近期动态。

## 信息不足时的处理

如果确实找不到公司准确信息，输出：

```markdown
## 已查到的信息
- 来源：
- 能确认的事实：

## 信息不足
- 未找到或无法确认：
- 不能下结论的部分：

## 基于 JD 的最低限度判断
- 岗位可能服务的对象：
- 岗位可能交付的结果：
- 候选人可连接的经历：

## 面试诚实表达
- 可说：
- 可问：
- 不要说：
```

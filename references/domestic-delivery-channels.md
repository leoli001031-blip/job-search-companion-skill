# 国内投递平台生态与差异化策略

用于 BOSS 直聘、智联招聘、猎聘、拉勾、前程无忧、实习僧、公司官网/公众号、内推等国内投递渠道。

## 核心原则

国内求职通常不是“投简历然后等”。不同平台有不同机制：

- BOSS 更偏即时沟通。
- 智联、前程无忧更偏传统投递和简历搜索。
- 猎聘更常出现猎头/顾问角色。
- 拉勾偏互联网/技术岗位。
- 实习僧偏实习和校招。
- 公司官网/公众号更像正式网申。

每个渠道都要准备两套材料：

```text
在线简历：用于平台搜索、系统筛选和初步沟通。
附件简历：用于下载、转发、业务负责人阅读和面试归档。
```

## 平台策略

| 渠道 | 机制 | 简历重点 | 沟通重点 |
| --- | --- | --- | --- |
| BOSS 直聘 | 直聊、匹配、在线沟通 | 在线简历完整，求职意向清楚，关键词要能搜到 | 打招呼短，先给匹配点，不写长求职信 |
| 智联招聘 | 投递 + HR 搜索 | 在线简历字段完整，附件 PDF 规范 | 投递后可简短补充亮点 |
| 猎聘 | 猎头/顾问参与较多 | 年薪口径、项目影响、岗位层级清楚 | 快速回应猎头，确认不要重复投递同公司 |
| 拉勾 | 互联网/技术垂直 | 技术栈、项目链接、可验证交付物 | 可以更技术化，但仍要讲业务结果 |
| 前程无忧 | 传统行业、外企、制造业常见 | 标准中文简历、时间线清楚 | 语气正式，附件命名规范 |
| 实习僧 | 实习/校招 | 教育、专业、可实习时长、到岗时间 | 明确实习周期和每周出勤 |
| 官网/公众号 | 正式网申 | 按字段填写，信息一致 | 保存投递记录和岗位编号 |

## 投递时间窗口

经验性建议，不是硬规则：

- 优先：工作日 9:00-11:00、14:00-16:00。
- 谨慎：深夜、凌晨、周末集中海投，容易被淹没，也显得不够职业。
- BOSS/直聊类平台更看重对方在线状态；如果对方刚活跃，可以优先沟通。
- 官网/网申类平台不必过度纠结时间，重点是信息完整和岗位编号保存。

## 高价值岗位的狙击投递

不是所有岗位都值得重投入。对用户特别想去、岗位质量高、匹配度有机会、或有人脉入口的岗位，使用“狙击投递”而不是海投。

适用条件：

- 用户明确想去这家公司/岗位。
- JD 信息较清楚，能拆出 5-8 道核心题。
- 有公司名、业务线或可识别产品，能做公司理解。
- 用户有至少 1 个可展示证据，或愿意先做 1 个短期作品/模拟案例。

流程：

```text
锁定 3-5 个高价值目标
-> 做公司理解卡和 JD 题目清单
-> 找到可能的信息入口：内推人、校友、前员工、公开访谈、官网/公众号/招聘页
-> 准备 2-3 个具体问题，了解岗位真实工作
-> 准备主动提案：我可以先帮你做 A/B/C
-> 发送短沟通钩子 + 简历 + 作品/案例/提案页
-> 跟进一次，有反馈就迭代材料
```

高价值投递包：

```text
1. 定制简历
2. 一句话匹配理由
3. 作品/案例链接或 1 页 PDF
4. 公司理解卡或岗位提案页
5. 可联系时间
```

努力信号不是堆字数，而是让对方看到：

- 你真的研究过这家公司和岗位。
- 你能回答 JD 的关键题。
- 你有可展示作品，而不是只会口头承诺。
- 你能主动提出具体帮法，不需要对方替你设计工作。

## BOSS 打招呼

不是求职信。目标是让对方愿意点开简历。

结构：

```text
岗位相关背景 + 1 个证据点 + 希望沟通
```

模板：

```text
您好，我关注 [岗位方向]。我有 [相关经历/行业背景]，也做过 [可证明项目/作品/成果]，和岗位里的 [核心任务 1]/[核心任务 2] 比较匹配，想进一步沟通。
```

规则：

- 控制在 30-80 字。
- 不重复粘贴完整自我介绍。
- 不一上来问加班、五险一金、薪资细节；这些放到匹配确认后。
- 已读不回是常见现象，不代表最终拒绝。

## BOSS 简历 + 作品/案例链接组合

如果候选人有项目网页、作品集、案例页、作品链接或可展示成果，并计划和简历一起在 BOSS 发送，采用“短钩子 + 简历 + 作品/案例链接”的组合。

原则：

- 简历负责第一眼匹配：岗位方向、教育/经历、核心项目摘要、关键词。
- 作品/案例链接负责第二层证明：截图、录屏、输入输出案例、业务场景、结果指标或可验证交付物。
- 不要只发裸链接。必须用 1 句话说明链接里有什么、为什么和岗位相关。
- 如果平台限制外链或 HR 不方便打开，准备 PDF 版项目案例、截图长图或 3 张关键截图作为替代。
- 链接首屏必须让 HR 3 秒内看懂：候选人姓名、目标方向、核心作品/案例、能证明什么岗位能力。

组合话术：

```text
您好，我关注 [岗位方向]。简历已发，另外附了 [作品集/项目页/案例页]，里面有 [1-3 个与岗位相关的成果]，能更直观看到我在 [岗位核心能力] 上的实践，方便的话想进一步沟通。
```

更短版本：

```text
您好，简历和 [作品/案例链接] 一起发您了。链接里有 [与岗位相关的成果/项目/作品]，和岗位里的 [核心任务] 比较相关，想争取一次沟通机会。
```

作品/案例链接最低要求：

```text
候选人是谁
投什么方向
核心作品/项目/案例分别证明什么能力
每个核心案例至少 1 个截图、样例、数据、链接或可验证交付物
Demo / 录屏 / 作品 / GitHub / 报告等入口
联系方式
```

## 已读不回是常态

BOSS/直聊平台的“已读不回”通常只能说明没有进入即时沟通，不等于正式拒绝。常见原因：

- HR 同时处理大量候选人，只回复优先级最高的一部分。
- 岗位不急，先收集简历。
- 候选人进入待比较池，HR 等业务方反馈。
- 已读状态可能不等于认真阅读。

处理方式：

- 当作“无信号”，不要情绪化。
- 1 次轻跟进即可，跟进必须提供增量信息。
- 仍无回复时转向下一个机会，不要反复追问。

## 跟进节奏

```text
未读：等 2-3 个工作日。
已读未回：补一条有增量的信息，例如作品链接/项目截图/到岗时间。
对方提问：尽量当天回复。
面试后：24 小时内可发简短感谢和补充材料。
```

跟进模板：

```text
补充一下，我整理了一个 [项目截图/作品链接/案例页]，能更直观看到我在 [岗位相关能力] 上的实践。如方便，欢迎继续沟通。
```

不要只发：

```text
在吗？
有结果了吗？
怎么不回？
```

## 在线简历关键词

如果用户用 BOSS、智联、猎聘等平台，必须检查在线简历：

- 求职意向是否和附件简历一致。
- 工作经历和项目经历是否含 JD 关键词。
- 技能区是否有可搜索词，而不是只有长段叙述。
- 每段经历是否有 1-2 个可被搜索/筛选的岗位关键词。

在线简历压缩版 bullet：

```text
完成 [项目/作品/任务]，覆盖 [关键流程/核心动作]，产出 [交付物/结果/证据]，可通过 [链接/截图/报告/Demo] 验证。
```

## 平台曝光优化

平台算法不公开且会调整，以下是经验性建议：

- 在线简历完整度越高，越容易被系统推荐或被 HR 搜到。
- 岗位名、技能、行业、城市、到岗时间、期望薪资不要乱填“不限”。
- 工作/项目经历中要出现 JD 关键词，但必须能解释。
- 求职状态和活跃度会影响部分平台曝光；求职期保持资料更新。
- 不要频繁海投完全不匹配岗位，可能降低账号质量或沟通转化。
- 打招呼额度有限的平台，要优先投匹配度高、近期活跃、JD 清楚的岗位。

## 同公司多平台投递冲突

同公司同岗位不要在 BOSS、官网、内推、猎头等渠道重复乱投。可能出现：

- HR 系统去重后只保留最早版本。
- 猎头和内推渠道冲突。
- 不同版本简历内容不一致，降低可信度。

建议：

```text
先记录岗位编号和投递渠道。
优先选择内推/官网/目标平台之一。
如果换渠道，确保简历版本、岗位名、时间线一致。
```

## 公司官网 / 公众号网申

注意：

- 保存岗位链接、岗位编号、投递日期。
- 字段信息必须和附件简历一致。
- 官网系统不一定支持复杂附件，准备纯文本版经历。
- 网申后不要只等系统，可同步找内推或平台沟通。

## 内推入口

内推请求不要只说“帮我内推一下”。至少给：

```text
岗位链接
PDF 简历
一句话匹配理由
可联系时间
```

详细见 `domestic-referral-mechanism.md`。

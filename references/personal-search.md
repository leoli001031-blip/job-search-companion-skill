# 个人信息搜索

用于从原始材料中建立候选人画像。

## 开放式启动

用户可以用任何方式开始：丢一个文件夹、一份旧简历、几个链接、项目截图、作品集、聊天式自述，或者只说“我想找工作但不知道怎么整理”。不要要求用户先选择固定模式。

AI 的启动动作：

1. 先接收和梳理用户已经给出的材料。
2. 从材料里提取候选人画像、可用证据、缺失信息和明显冲突。
3. 如果材料不足，先用已有内容推进一版初步判断，再追问最关键的缺口。
4. 如果用户完全没有材料，用轻量访谈问题开始，不要一次性发长问卷。

如果不同材料里的日期、岗位名称、学历、公司名互相冲突，先停止写作，要求用户确认正确版本。

启动阶段不是只让用户“发材料”。可以同时做两条线：

```text
材料收集：用户已有的文件、链接、项目、公开页面
主动追问：用户能直接回答但文件里没有的信息
```

具体问题清单见 `references/intake-questions.md`。候选人素材类型见 `references/candidate-materials.md`。问题清单是备选工具，不是强制问卷；每轮只问最关键的问题。用户回答不了的，标记为“待补”，不要阻塞整个流程。

## 搜索范围

只搜索求职相关来源：

- 本地简历、笔记、PDF、文档、图片、截图、项目 README、作品集。
- 工作/实习证明、获奖证书、竞赛材料、成绩单、课程项目、毕业设计、论文、活动照片、PPT、作品集。
- 公开 GitHub / GitLab 仓库、个人网站、文章、视频、演讲、Demo、内容账号。
- 用户提供的 LinkedIn、BOSS/猎聘/智联等平台资料、作品集、文章、证书、工作经历材料。
- 用户口头回答的求职偏好、项目背景、成果数字、限制条件。

读取本地文件夹前，先向用户说明本轮会查看的范围和目的，例如“只看简历、作品集、项目说明、证书和岗位相关文档”。不要主动展开聊天记录、财务材料、证件照片、私人相册、浏览器数据、无关下载目录等明显隐私内容。

避免无关隐私搜索。不要把网上搜到的同名结果直接当成候选人经历。

## 搜索问题

写任何重要材料前，先回答：

0. 用户已经提供了什么？当前目标、隐私边界和最急的问题是什么？
1. 用户当前想投哪些岗位、行业、城市或机会类型？有哪些硬性限制？
2. 哪些经历能解释候选人的职业路径？
3. 已经有哪些项目、产出、公开证明？
4. 哪些技能有证据，不只是自称？
5. 哪些数字、截图、证书、作品、Demo、链接、评价能支撑关键表述？
6. 投递前还需要补哪些证据？
